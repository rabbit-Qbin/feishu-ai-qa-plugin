# AI 问答插件 - 需求与优化清单

根据《项目总结文档》和本仓库文档整理，便于后续开发对照。

---

## 一、已实现功能（当前能力）

| 功能 | 说明 |
|------|------|
| **三阶段智能分析** | 用户提问 → 意图识别（是否要读数据）→ 查询计划生成（AI 决定读哪些字段/条件）→ 飞书 SDK 读数据 → AI 基于数据生成回答 |
| **多维表格选择** | 支持选择不同多维表格作为数据源 |
| **Create/Config/View** | 符合飞书应用插件三种状态 |
| **全量字段读取** | 为保障分析质量，会读取所有字段（如 500+ 条记录） |
| **中断请求** | 使用 `AbortController` 实现「停止生成」 |
| **回车发送** | Enter 发送，Shift+Enter 换行 |
| **拖拽文本** | 支持把文字拖进输入框 |

---

## 二、技术栈（文档记录）

- **前端**: TypeScript + Vite
- **飞书**: @lark-base-open/js-sdk
- **AI**: Moonshot (Kimi) — `moonshot-v1-8k`，端点 `https://api.moonshot.cn/v1/chat/completions`
- **部署**: GitHub Pages + GitHub Actions（已修复 package-lock、environment、权限）

---

## 三、文档里写的「未来优化方向」

来自《项目总结文档》§ 未来优化方向：

1. **弹窗模式**：将 AI 问答界面改为弹窗形式，提升体验  
2. **API Key 管理**：把 API Key 移到配置中，支持用户自定义（当前为硬编码）  
3. **数据缓存**：对频繁查询的数据做本地缓存  
4. **流式响应**：支持 AI 回答流式输出，提升首字/首句响应速度  
5. **历史记录**：保存用户问答历史，支持查看与重新提问  

---

## 四、注意事项（文档提醒）

- API Key 安全：生产环境应用环境变量或配置管理，不要硬编码  
- 数据量：飞书 SDK 单次最多 500 条，需分页  
- 部署后若看到旧版本：强制刷新（Ctrl+Shift+R）  
- 飞书返回数据类型需做安全转换（如 `toText()`）
